# Coding Conventions

- Write production-ready code that is clean, structured, and maintainable.
- Prefer small, reusable functions and modules with single responsibilities.
- Use clear, descriptive names for files, classes, methods, and variables.
- Keep code consistent with the existing project style.
- Add type hints where appropriate.
- Write docstrings for public functions, classes, and modules when helpful.
- Avoid duplication; extract shared logic into helpers or services.
- Keep functions short and easy to test.
- Follow best practices for logging, validation, and error handling.
- Do not introduce unnecessary complexity.
