# Architecture Rules

- Follow a layered and modular structure.
- Separate API, service, repository, and utility logic clearly.
- Keep business logic out of route handlers and UI components.
- Put configuration in dedicated files or modules.
- Use dependency injection or clean abstraction where useful.
- Keep components decoupled and easy to replace.
- Reuse existing patterns in the repository before introducing new ones.
- Prefer scalable structure over quick one-off solutions.
- Keep the codebase easy to navigate for future developers.
